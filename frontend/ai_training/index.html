RAG / SFT 資料製作指引

本分頁不會生成、審核、儲存或發布任何資料。管理員須根據已核實的來源，以人手建立 JSONL 檔案，再按照 Workstation 的發布流程進行驗證、簽署及安裝。

先了解 RAG 與 SFT 的分別

  • RAG 儲存需要檢索、引用、更新或撤回的知識。 事實、規則、案例及引用資料應存放於 RAG，不應依靠模型參數記憶。
  • SFT 教授模型如何作答，例如語氣、結構、分析步驟及拒答方式。 不應用作背誦會改變或需要引用的知識。

開始前，請先按 Google 文件內的提交紀錄確認來源已完成匿名化、具備使用權，而且內容仍然有效。 請保留來源擁有人、版本／日期、授權依據、內容 SHA-256 及撤回聯絡方法。

RAG 資料製作流程

  1. 根據規則、教練筆記、已授權案例或可靠參考資料建立 canonical 原文。不得將 AI 生成的摘要視為事實來源。
  2. 按語義分段：每段只處理一個主題，並須在離開原文件後仍可獨立理解。 不得將完整長文直接視為一個段落，亦不得為增加數量而重複近似段落。
  3. 建立 documents.jsonl。每行只可包含一個 JSON 物件，亦即 每行為一個 JSON 物件，而且只可包含 "id"、"title"、"text"、 "source_url"。
  4. id：必填字串，必須穩定且唯一，而且不得在整個檔案內重複，例如 fictional-library-hours-2099-001。同一段資料更新時應建立新版本 ID， 不得以行號作 ID。
  5. title：必填字串，須列明主題、資料類型及版本／年份，讓檢索結果 離開原文件後仍可辨識。
  6. text：必填字串，保存可直接引用的完整段落，包括規則、適用範圍、 例外及必要限制。不得只填關鍵字、網址或未經核對的 AI 摘要。
  7. source_url：必須存在的字串欄位。公開來源須填不含帳號、密碼或 fragment 的 HTTPS 網址。純內部或非公開資料可填空字串 ""， 但來源擁有人及授權證據仍須保存在管理紀錄。 
{"id":"fictional-library-hours-2099-001","title":"虛構示例｜星河中學圖書館延長開放時間｜2099","text":"本段為虛構示例。星河中學在 2099 年校內模擬辯論中，假設圖書館逢星期五延長開放一小時。此設定只供示範 JSONL 格式，並非真實校規。","source_url":""}

完成後，請將 documents.jsonl 放置於 RAG bundle 根目錄，建立新版本及 manifest，核對檔案雜湊值（hash），再按照 Workstation runbook 進行離線簽署。舊版本不得原址改寫。

SFT 資料製作流程

建議檔名為 sft.jsonl，方便人手辨識。現時未有 Workstation SFT 匯入器，因此這個名稱並非程式強制的 loader contract。

  1. 每行只可包含一個 JSON 物件。最外層只可包含 messages， 其值必須為陣列。本項目目前每行固定依次放置一個 system、 一個 user 及一個 assistant 訊息，並僅訓練 assistant 回覆。
  2. 每個訊息物件只可包含 role 及 content。 role 必須是上述三個角色之一，content 必須為 非空白字串。
  3. system 寫共同角色、語氣、處理資料不足及安全規則，不應放入 會改變的賽規或個別個案事實。user 寫真實、完整且可回答的任務， 並加入答案確實需要的上下文。
  4. assistant 寫經辯論訓練人員逐字審核的理想答案，以示範預期語氣、 結構、分析步驟、引用方式及資料不足時的處理方法。不得加入未核實事實。
  5. Google 文件內的「提交者」、「標題」及「來源」只供授權、追溯及撤回管理， 不應寫入 SFT JSONL。若任務本身需要來源內容，應把已核對且必要的上下文寫入 user.content，而可更新知識仍應放入 RAG。
  6. 原始講稿或逐字稿並非天然的問答配對。須先由人手界定問題、所需上下文及理想答案， 不得將相鄰段落自動視為 user／assistant。
  7. 同一來源、同一場比賽或同一份稿件只可歸入一個 split，避免內容同時出現於 train、validation 及獨立 eval。 
{"messages":[{"role":"system","content":"你是虛構校園的辯論練習助手。請以香港正體中文提供清晰、具體的分析，資料不足時須明確說明。"},{"role":"user","content":"以下是虛構辯題：星河中學應否在星期五延長圖書館開放時間？請示範如何比較正反雙方影響。"},{"role":"assistant","content":"這個虛構辯題可以從受益人數、影響程度、執行成本及替代方案四方面比較。正方須證明延長時間能實際改善學生溫習需要，反方則須交代人手及保安成本是否足以抵銷效益。最後應比較哪一方的影響較廣泛、較持久，而且更難由其他措施取代。"}]}

發布前檢查

  • 逐項核對授權、匿名化、來源、日期、適用範圍、事實準確度及撤回狀態。
  • 移除重複內容、互相矛盾但沒有版本標示的內容、個人資料及無法追溯來源的主張。
  • 驗證每行 JSON 均可獨立解析，ID 沒有重複，檔案雜湊值、版本及 manifest 一致。
  • 預留完全未參與製作的評估案例（eval cases），以測試檢索命中、引用準確度、拒答及私隱處理。
  • 只有在 blind eval 通過後，方可簽署及安裝新 bundle。如未能通過，則保留現行 active 版本。
