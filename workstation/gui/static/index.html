硬件及服務

模型／RAG 資料

只有你先查看 signed inventory，再明確確認，系統先會 pull code allowlist 內模型；完成後會逐一核對 signed digest。RAG 會驗簽及 hash、在新目錄建立 index，成功後才原子切換。

        
程式更新／Rollback
每次更新都會先 Drain，驗 Ed25519 簽名、hash 及相容性，再經完整 health gate；失敗會自動切回。

        
GPT-SoVITS 資料準備／訓練
先經 Tailscale SSH 將一個 recordings.json 或 ZIP 放入 /srv/lmc-ai/datasets/incoming，檔名必須等於 dataset ID。訓練不會自動啟用新聲線。
